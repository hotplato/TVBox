# 多仓配置支持设计

**日期**：2026-07-14  
**状态**：已确认

## 目标

支持社区标准多仓 JSON（`urls: [{ name, url }]`）：用户配置一个多仓地址后可选子仓加载；记住上次选择，设置页可切换。

## 方案

双层 URL（方案 A）：

| Hawk 键 | 含义 |
|---|---|
| `API_URL` | 用户配置的地址（多仓或单仓） |
| `STORE_API` | 当前选中子仓 URL |
| `STORE_NAME` | 当前选中仓名称（设置页展示） |

## 识别与加载

1. 拉取 `API_URL`
2. 若 JSON 含非空有效 `urls` → 多仓（优先于同文件中的 `sites`）
3. 若 `STORE_API` 仍在列表中 → 拉取该子仓并按单仓解析
4. 否则 → 主线程弹窗选仓，写入 `STORE_*` 后再拉子仓
5. 无 `urls` → 现有单仓 `sites` 解析

## UI

- 首次无有效记忆：首页 `SelectDialog`「请选择仓库」
- 设置页「配置地址」下增加「当前仓库」：有多仓时显示名并可切换；单仓显示「无」
- 变更 `API_URL`（含扫码）时清空 `STORE_*`

## 错误与边界

- 多仓/子仓拉取失败：现有重试对话框；子仓失败不清 `STORE_*`
- `urls` 无有效项：「多仓列表为空」
- 选仓取消：无记忆可重试；设置页取消保持原仓
- 不合并多仓、不做首页长按切仓

## 验证

- 多仓首次选仓、重启记忆、设置切换、改单仓/再改多仓、`assembleDebug`
