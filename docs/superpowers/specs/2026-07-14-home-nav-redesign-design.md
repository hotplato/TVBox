# 首页 / 我的 / 分类导航重设计

**日期**：2026-07-14  
**状态**：已确认

## 目标

改善 Compose 首页体验：消除「我的」入口与「分类」挤在同一左栏导致的拥挤、层次不清与遥控器焦点混乱；对齐经典 TVBox 的顶部分类浏览习惯。

## 已确认决策

| 项 | 选择 |
|---|---|
| 整体壳 | 顶部分类 Tab +「我的」独立页（方案 A） |
| 内容区 | 保持多行海报网格（约 5 列，纵向滚动） |
| 「我的」页 | 简洁纵向列表（历史 / 收藏 / 推送 / 设置） |
| 痛点优先级 | 左栏太挤、层次不清、焦点路径乱 |

## 信息架构

```text
首页
├── 顶栏
│   ├── 左：主源名称（可焦点；点击切源，对齐经典）
│   ├── 中：横向可滚动分类 Tab
│   └── 右：搜索 / 直播 / 我的
└── 主体：LazyVerticalGrid 海报（满宽）

我的（独立路由）
├── 返回
├── 历史记录 → History
├── 我的收藏 → Collect
├── 推送 → Push
└── 设置 → Settings
```

搜索、直播仍从首页顶栏进入；不进入「我的」列表，避免与顶栏重复。

## UI 规格

### 首页

- 去掉当前 220dp 左栏（「我的」按钮组 + 纵向分类 `LazyColumn`）。
- 顶栏单行：源名 + 分类 Tab + 右侧快捷入口。
- 分类切换：选中态高亮；横向滚动；选中后复用现有 `HomeViewModel.selectSort` / 列表加载逻辑。
- 主体：仍用现有 `LazyVerticalGrid` + `TvPosterCard`；loading / empty / error 状态保留。
- 仓库选择提示（`storePrompt`）流程不变。

### 「我的」页

- 新路由（如 `mine`），顶栏「返回」+ 标题「我的」。
- 纵向列表四项，复用 `TvFocusButton`（或同等焦点表面），无历史预览卡片、无四宫格。
- 子页（历史 / 收藏 / 推送 / 设置）内部 UI 与逻辑本迭代不改。

### 焦点

- 纵向两层：上=顶栏（源名 / Tab / 快捷），下=海报表。
- 同层左右移动；从内容区按上回到顶栏对应列。
- 「我的」返回首页后，焦点尽量落回「我的」按钮。

## 组件与文件

| 变更 | 说明 |
|---|---|
| `HomeScreen.kt` | 重做布局为顶栏 + 网格 |
| 新建 `TvCategoryTabRow`（或等价） | 横向分类 Tab，支持选中与焦点 |
| 新建 `HomeTopBar`（可内联） | 源名 + Tab + 快捷入口组合 |
| 新建 `MineScreen` + 路由 | 简洁列表页 |
| `TvBoxRoutes` / `TvBoxNavGraph` | 增加 `MINE`；首页 `onOpenMine` |
| `HomeViewModel` | 数据流尽量不动；若需切源再补 UI 触发 |

## 明确不做

- 不重做历史 / 收藏 / 设置 / 推送 / 搜索内部页
- 不改播放器、爬虫、`ApiConfig` 加载语义
- 不恢复左栏主导航（含窄主栏方案 B）
- 「我的」页不做最近观看预览大卡片

## 验收

1. 首页无左栏；分类在顶部横向切换，网格多行展示
2. 搜索 / 直播 / 我的从顶栏可达；我的页四项进入现有子页
3. 遥控器：顶栏 ↔ 网格焦点路径清晰，无明显死角
4. `./gradlew assembleDebug` 通过
